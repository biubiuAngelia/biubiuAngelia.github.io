相对路径和绝对路径

相对路径

相对路径是对于当前路径来说的，以当前路径为参照查找的路径，有的时候直接写绝对路径，路径比较深，要写的路径也比较长

绝对路径（无论在哪个页面，无论如何都会找到文件）

绝对路径是从根目录开始查找，跟当前文件没有关系

http

http是什么

HyperText Transfer Protocol ,超文本传输协议

http协议的作用

http协议主要用作浏览器和服务器之间的数据传输

http协议是无状态的

服务器并不会记住浏览器的状态，每次请求都会认为是新的浏览器在访问服务器

无状态的问题需要解决，否则很多效果没法实现（如：登录的效果）

cookie
session

如何通过cookie和session实现http会话

服务器端可以在session中存储浏览器的状态信息，这个session中存储的数据会具备一个session id，在服务器在响应请求的时候，将这个session id直接写入到浏览器端的cookie中，由于cookie在每次请求的时候，都会被自动的被请求携带到服务器端，所以每次请求这个session id 都会被拿到服务器端，那么服务器就可以通过浏览器发送过来的session id 来找，浏览器上一次的状态。

cookie

概念

浏览器端的一个存储空间。

特点

  1. cookie的存储是以域名为单位的，各个域名之间不能互相访问cookie。
  2. 每个域名能够在cooki中存储的内容是有大小限制的，4kb
    1kb=1024b ,一个汉字占2b一个英文字母占1b
  3. cookie中存储的数据是有时限的。（默认浏览器关闭，cookie消失）
  4. cookie在每个域名下是分路径存储的，在父级路径中是不能访问到存储在子级路径中的cookie，但是，子级路径可以访问到存储在父级路径中的cookie，所以一般情况下，都将所有cookie存储在根目录下，以便所有的路径都可以访问
    例如
    在http://studyit.com/danshboard/index 页面中存储了一个cookie，那么在http://studyit.com 这个页面就访问不到cookie了
  5. cookie存储的信息，会在每次请求的时候，会被携带者发送到服务器

跨域：当前所在的网页的域名，和要请求的数据的域名，不一样

由于浏览器的同源策略，域名中（协议、域名、端口）有任意一部分不一致，就会产生跨域请求。浏览器会直接将跨域请求阻止。

  1. 协议
  2. 域名
  3. 端口

跨域的解决方案

  1. jsonp（最常用）
    原理：通过动态创建script标签，将script标签的src属性设置为要请求的接口地址，这样将script加到页面上的时候，会自动向接口发送请求了。服务器端返回的数据会直接被当作js代码执行。

服务器端返回的数据，有格式的要求，必须是一个函数调用的形式
函数名(数据)
这个函数可以在请求之前就在全局之前创建号，那么请求完成之后，会自动调用这个函数，就类似于请求成功的回调函数了。

  1. 后台在相应头中添加一个 Access-Control-Allow-Origin:*
