前端模块化

模块化

说的就是对于js代码的管理。将指定功能的内容划分为一个个的模块，这些模块在使用的时候可以被引用进来。有复用性，并且代码分门别类，容易维护和管理。

CommonJS 模块化标准

CommonJS 通用模块定义（后端），主要用在Node.js当中,代表就是Node.ks

AMD 也是一个模块化标准

Async Module Definition 异步模块定义，适用于前端开发，代表Require.js
基于CommonJS衍生出来的（前端）

CMD 也是一个模块化标准

Common Module Difinition 适用于前端开发，代表sea.js

AMD与CMD区别

AMD 所有的模块尽可能早的会被加在进来
CMD as lazy as possible，只有需要的时候才会加载（会有一定的bug）

require.js没有明显的bug,sea.js明显没有bug

require.js的学习

requirejs 它是帮我们实现前端模块的工具，它遵循的标准AMD

定义模块

  1. requirejs提供了一个全局的函数，来定义模块difine()

  2. 一个模块通常都会单独保存成一个文件

  3. 当模块要提供内容给别人的时候，需要使用returne语句将内容进行返回
    当模块依赖于别的模块的时候，需要给当前模块添加依赖项

引用模块

  1. requirejs提供了一个去哪儿去的函数。来引用模块require()。
  2. 第二个参数的函数，为模块加在完毕之后的回调函数
  3. 如果引用的模块有产出内容，在引用模块内通过returned返回了东西，我们可以通过在回调函数中添加形参，来接收前面模块返回的内容，行参合模块是一一对应的
    一般情况下，会将有返回值的全部放在前面，因为这样可以省略掉后面没有返回值的模块的形参

data-main属性的说明

  • 在请求requirejs的script标签中添加一个data-main属性，指向一个js文件
  • 当requirejs加载完成之后，会直接去执行这个data-maind属性所指向的js文件的内容
  • data-main所指的文件是异步加载的，所以如果在下面有依赖于data-main文件中实现的功能的代码，那么，很可能出错

使用difine()方法定义一个模块

在请求requirejs的script标签中添加一个data-main属性，指向一个js文件，当require夹在完成之后,会直接去执行这个data-main属性所指向的js文件的内容

data-main所指向的文件是异步加载的，所以如果下面有依赖于data-main文件中实现的功能的代码，很有可能会出错。

当直接使用require来引用一个模块的时候，那么模块的路径会以当前调用require方法的html页面为基础来查找
如果使用了data-main属性，路径的查找回忆这个data-main所指的文件为基础。

require.config方法的学习

使用require加载第三方不支持模块化的内容

  • config方法可以对requirejs进行配置
  • 传的参数是一个对象

    模块路径的研究

  1. 如果使用basaUrl来设置基本路径，那么所有的模块查找都会以改路径作为基础
  2. 如果按照baseUrl找不到文件，那么require还会以引用模块的代码所在所在的文件为基础继续查找
  3. 如果通过config方法指定了baseUrl，那么所有的模块查找都会以baseUrl中设置的路径为基础
    注意：在设置了baseUrl之后，文件会首先按照baseUrl作为基础去查找，如果找到了，就直接使用
