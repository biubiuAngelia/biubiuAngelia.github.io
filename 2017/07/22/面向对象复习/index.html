面向对象三大特征

封装

对象就是对属性和方法的封装，要实现一个功能，对外值暴露一些接口，调用者只需要通过接口调用功能即可，不需要关注接口内部实现原理：
Math.abs();
方法和函数：对象内部的函数叫做方法

继承

子承父业
一个对象有很多属性和方法，另一个对象也想有这些属性和方法

多态

同一个类型的变量可以表现出不同的形态
用父类的变量指向子类的对象
js中没有多态

创建对象的方式

字面量

Object构造函数

代码复用率太低

自定义构造函数

提高代码复用性，可以快速创建多个对象

构造函数的语法

  1. 构造函数的函数名首字母大写，帕斯卡命令
  2. 构造函数一般和new关键字一起使用
  3. 构造函数不需要设置return 语句，默认返回的是新创建的对象
  4. this指向的是新创建的对象

构造函数的执行过程

  1. new关键字，会创建一个心的对象，会在内存中开辟一个新空间
  2. 让构造函数中this指向新创建的对象
  3. 执行构造函数，给新创建的对象进行初始化，赋值
  4. 构造函数初始化完成后，会将新创建的对象返回

构造函数注意点

构造函数本身也是函数

  1. 构造函数有返回值
    如果return 返回的值是值类型的数据，则对构造函数没有影响
    如果return 返回的值，是引用类型的数据，会用 引用类型的数据 替换 新创建对象
  2. 构造函数当作普通函数执行会怎样

构造函数的问题

  • 构造函数每执行一次，都会为对象开辟一个新的空间，在对象中，所有对象的方法代码都是一样的，没有必要开辟新空间，造成内存浪费
    构造函数—->原型

原型

  • 每个构造函数在创建出来的时候，系统会自动给这个构造函数创建并关联一个空的对象，这个空对象就叫做原型
  • 在原型中定义的属性和方法，会被这个构造函数创建出来的对象所共享
    访问原型的方式：构造函数名.prototype

原型的使用方式

举例：原型添加方法给

原型的注意点

  1. 使用【对象.属性名】去获取对象的属性时，会先在自身中进行查找，如果没有，就去原型中查找
  2. 使用【对象.属性名】去设置对象属性的时候，只会在自身进行查找，如果有，就去修改，如果没有，就添加
  3. 一般情况下，不会将属性放在原型当中，只会将方法放在原型中
  4. 在替换原型的时候，替换之前创建的对象，和替换之后创建的对象的原型不一致！
  5. 对象关联的原型，就是创建对象那一刻，构造函数的prototype属性

概念

属性的搜索原则

当通过对象名.属性名获取属性是 ，会遵循以下属性搜索的原则：
1-首先去对象自身属性中找，如果找到直接使用，
2-如果没找到去自己的原型中找，如果找到直接使用，
3-如果没找到，去原型的原型中继续找，找到直接使用，
4-如果没有会沿着原型不断向上查找，直到找到null为止
