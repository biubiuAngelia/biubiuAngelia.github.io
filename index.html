必填

需要指定表单验证的规则，就得给表单元素添加对应的data-属性

  1. data-required:设置当前表单项为必填项
  2. data-trim:验证的时候是否要去掉两边的空格（注意：提交数据的时候，空格不会被去掉）
  3. data-pattern:指定正则表达式来测试表单的值是否能够通过验证(不用加两边的”/“)

  1. data-conditional:指定条件验证的条件函数的名称，条件其实就是function
    在表单验证的hi后，会自动调用这个函数，函—–缺省—–
    条件验证找函数是去validate方法参数对象中找conditional对象
    在conditional对象中找名字相同的function

  2. data-description

  3. data-describedby

可以给validate方法传递的参数

  1. conditional:它是一个对象，条件验证的所有的函数都会存放在这个函数中
  2. description:是一个对象，存储的是一组组（对象）描述信息
  3. onBlur:布尔值，设置是否在表单元素失去焦点的时候校验，默认false
  4. onChange:布尔值，设置是否在表单内容改编的时候进行校验,默认false（触发了验证，但不改变提示信息）注意：只有在失去焦点的时候校验
  5. onKeyup:布尔值，设置是否在按键弹起的时候进行校验，默认false（触发了验证，但不改变提示信息）
  6. onSubmit:布尔值，设置是否在表单提交的时候校验，默认true
  7. sendForm:布尔值，设置是否在验证成功之后提交表单，默认true

回调函数事件

  1. valid:当表单整体通过验证的时候，会调用该方法
  2. invalid:当表单整体不通过验证的时候，会调用该方法
  3. eachField:每当有一个字段被校验的时候，就会调用该方法（不管校验通过不通过）
  4. eachValidField:每当有一个字段被检验且通过验证的时候，调用该方法
  5. eachInvalidField:每当有一个字段被校验且校验不通过的时候，调用该方法

构造函数的语法

  1. 构造函数的函数名首字母大写，帕斯卡命令
  2. 构造函数一般和new关键字一起使用
  3. 构造函数不需要设置return 语句，默认返回的是新创建的对象
  4. this指向的是新创建的对象

构造函数的执行过程

  1. new关键字，会创建一个心的对象，会在内存中开辟一个新空间
  2. 让构造函数中this指向新创建的对象
  3. 执行构造函数，给新创建的对象进行初始化，赋值
  4. 构造函数初始化完成后，会将新创建的对象返回

构造函数注意点

构造函数本身也是函数

  1. 构造函数有返回值
    如果return 返回的值是值类型的数据，则对构造函数没有影响
    如果return 返回的值，是引用类型的数据，会用 引用类型的数据 替换 新创建对象
  2. 构造函数当作普通函数执行会怎样

构造函数的问题

  • 构造函数每执行一次，都会为对象开辟一个新的空间，在对象中，所有对象的方法代码都是一样的，没有必要开辟新空间，造成内存浪费
    构造函数—->原型

原型

  • 每个构造函数在创建出来的时候，系统会自动给这个构造函数创建并关联一个空的对象，这个空对象就叫做原型
  • 在原型中定义的属性和方法，会被这个构造函数创建出来的对象所共享
    访问原型的方式：构造函数名.prototype

原型的使用方式

举例：原型添加方法给

原型的注意点

  1. 使用【对象.属性名】去获取对象的属性时，会先在自身中进行查找，如果没有，就去原型中查找
  2. 使用【对象.属性名】去设置对象属性的时候，只会在自身进行查找，如果有，就去修改，如果没有，就添加
  3. 一般情况下，不会将属性放在原型当中，只会将方法放在原型中
  4. 在替换原型的时候，替换之前创建的对象，和替换之后创建的对象的原型不一致！
  5. 对象关联的原型，就是创建对象那一刻，构造函数的prototype属性

前端模块化

模块化

说的就是对于js代码的管理。将指定功能的内容划分为一个个的模块，这些模块在使用的时候可以被引用进来。有复用性，并且代码分门别类，容易维护和管理。

CommonJS 模块化标准

CommonJS 通用模块定义（后端），主要用在Node.js当中,代表就是Node.ks

AMD 也是一个模块化标准

Async Module Definition 异步模块定义，适用于前端开发，代表Require.js
基于CommonJS衍生出来的（前端）

CMD 也是一个模块化标准

Common Module Difinition 适用于前端开发，代表sea.js

AMD与CMD区别

AMD 所有的模块尽可能早的会被加在进来
CMD as lazy as possible，只有需要的时候才会加载（会有一定的bug）

require.js没有明显的bug,sea.js明显没有bug

require.js的学习

requirejs 它是帮我们实现前端模块的工具，它遵循的标准AMD

  3. 当模块要提供内容给别人的时候，需要使用returne语句将内容进行返回
    当模块依赖于别的模块的时候，需要给当前模块添加依赖项

    一般情况下，会将有返回值的全部放在前面，因为这样可以省略掉后面没有返回值的模块的形参

data-main属性的说明

  • 在请求requirejs的script标签中添加一个data-main属性，指向一个js文件

http是什么

http协议的作用

http协议主要用作浏览器和服务器之间的数据传输

http协议是无状态的

服务器并不会记住浏览器的状态，每次请求都会认为是新的浏览器在访问服务器

无状态的问题需要解决，否则很多效果没法实现（如：登录的效果）

如何通过cookie和session实现http会话

服务器端可以在session中存储浏览器的状态信息，这个session中存储的数据会具备一个session id，在服务器在响应请求的时候，将这个session id直接写入到浏览器端的cookie中，由于cookie在每次请求的时候，都会被自动的被请求携带到服务器端，所以每次请求这个session id 都会被拿到服务器端，那么服务器就可以通过浏览器发送过来的session id 来找，浏览器上一次的状态。

cookie

概念

浏览器端的一个存储空间。

特点

  1. cookie的存储是以域名为单位的，各个域名之间不能互相访问cookie。
  2. 每个域名能够在cooki中存储的内容是有大小限制的，4kb
    1kb=1024b ,一个汉字占2b一个英文字母占1b
  3. cookie中存储的数据是有时限的。（默认浏览器关闭，cookie消失）
  4. cookie在每个域名下是分路径存储的，在父级路径中是不能访问到存储在子级路径中的cookie，但是，子级路径可以访问到存储在父级路径中的cookie，所以一般情况下，都将所有cookie存储在根目录下，以便所有的路径都可以访问
    例如
    在http://studyit.com/danshboard/index 页面中存储了一个cookie，那么在http://studyit.com 这个页面就访问不到cookie了
  5. cookie存储的信息，会在每次请求的时候，会被携带者发送到服务器

跨域：当前所在的网页的域名，和要请求的数据的域名，不一样

由于浏览器的同源策略，域名中（协议、域名、端口）有任意一部分不一致，就会产生跨域请求。浏览器会直接将跨域请求阻止。

  1. 协议
  2. 域名
  3. 端口

服务器端返回的数据，有格式的要求，必须是一个函数调用的形式

分支

分支可以理解为版本库的备份
在版本库中默认有一个主分支，我们可以通过git命令来创建一个新的分支，这个新的分支上面会有当前这个分支上所有的内容，也就类似于是做了一个备份出来。

分支的操作

  1. 如何创建分支
    git branch 分支名称
  1. 如何切换分支
    默认情况下，我们处在主分支下面，要在其他分支下做开发，首先要切换到该分支

git checkout 分支名

  1. 如何合并分支
    将指定的分支合并到当前分支上！
    git merge 要合并的分支名称

  2. 删除分支
    git branch -d 分支名称

分支的使用场景

多任务并行开发的时候，分支可以做到多个任务之间不会相互影响！

每当要进行一个新任务的开发的时候，都需要创建一个分支出来，单独做开发，功能完成之后，再将分支合并回去。

注意

主分支（master）是在第一次提交完成之后，才有的！！

每次切换分支之前，要将所有的内容做一次提交！！！否则可能丢失修改！！

任意分支都可开辟新的分支，新创建出来的分支，会拥有当前分支中所有的已有内容！

快捷创建并切换分支的命令

git chekcout -b 分支名称

回滚（回退到某个版本）之后，如果后悔了，不想回退了

通过git log -g 查看所要恢复的版本号（短期内可操作）

git branch 分支名 版本号

合并分支！

远程仓库

概念

远程仓库，主要用来，做多人协作开发的时候，代码库的共享工作！

gitHub 就是一个远程仓库！

远程仓库

1. 创建一个远程仓库

创建远程仓库的时候，文件夹名称一般都会用.git来结尾，表示当前文件夹是一个git库！

远程仓库其实就是一个裸仓库，裸仓库，就是直接将.git文件夹中的所有的内容直接暴露出来，不会再有.git文件夹了!

远程仓库的唯一的作用，就是用来做版本库共享。不能再远程仓库中做开发！

远程仓库创建好之后，一般就不会再有别的操作了！

git init –bare

2. 将本地仓库推送到远程仓库中

git push 远程仓库的地址 本地分支名称:远程分支名称

如果本地分支的名称好远程分支的名称一模一样那就可以只写一个分支名
例如： git push https://github.com/avakpan/frontend9.git master

3. 从远程仓库获取版本库到本地

将远程仓库的内容拷贝一份放到指定的本地文件夹当中，如果不指定本地文件夹名称，git会自动创建一个和远程仓库文件夹名称相同的文件夹

git clone 远程仓库地址 本地文件夹名称

4. 从远程仓库获取更新的内容

git pull 远程仓库地址 分支名称

5. 原则

一般情况下 需要向远程仓库推送新的内容之前，需要先获取一次最新的代码
执行push之前要执行一次pull操作

远程仓库刚创建出来的时候是一个空仓库

这时，可以将任意本地仓库通过push命令推送到远程仓库中，但是一旦远程仓库被push过一次之后，仓库就不再是一个空仓库了，之后能够被push到这个仓库中的内容，都是和仓库相关的，不能是任意一个好不相关的仓库，如果使用一个毫不相关的仓库进行push，会被拒绝！

远程地址管理

展示所有的别名

git remote

注意： 如果当前项目是通过git clone从远程仓库获取下来的，那么会默认的有一个远程地址的别名(origin)，指向的就是当时clone这个项目的时候的远程地址

origin https://github.com/avakpan/frontend9.git

添加远程地址别名

git remote add 别名 远程地址

删除远程地址

git remote remove 别名

SSH协议

Secure Shell 安全的shell

http协议的格式

http://域名:端口号

ssh 协议格式

用户名@域名

ssh 一般用作服务器的远程管理，我们可以通过ssh安全的shell管理服务器

在bash中提供了一个ssh命令，这个命令就是可以通过ssh协议连接服务器进行管理的！

ssh命令的说明

ssh 用户名@域名

以指定的用户名使用ssh协议登录域名对应的远程服务器

ssh免密登录的原理

在服务器端，有一个名单，名单中存储所有能够登录当前服务器的客户端的公钥！

由于每次客户端在请求登录的时候，会自动将客户端本身的公钥自动发送给服务器，服务器在拿到公钥之后，和名单中的内容进行一一对照，如果有当前这个公钥，那么就直接允许登录了！

在自己电脑上生成 公钥以及私钥！

ssh-keygen -t rsa

执行这个命令的时候，一路默认，一路敲回车
直到看到冒泡泡为止！

生成的秘钥默认在 用户根目录下面的.ssh文件夹中

c:\users\你自己的用户名.ssh

实现服务器免密登录的步骤（了解）

  1. 如果当前电脑从未生成过秘钥，要执行
    ssh-keygen -t rsa
    生成秘钥

  2. 在用户根目录下面的.ssh文件夹中找到生成密钥，id_rsa.pub就是我们需要的公钥文件，复制里面所有的内容

  3. 登录到服务器（这次登录还是要密码的），在用户根目录下面找到.ssh文件夹

  4. 打开.ssh文件夹中的 authorized_keys文件

  5. 将刚才复制的所有的公钥的内容 粘贴到该文件最后，保存退出

  6. 再次登录服务器，便不再需要密码了！！

实现github免密登录的过程

  1. 如果当前电脑从未生成过秘钥，要执行
    ssh-keygen -t rsa
    生成秘钥

  2. 在用户根目录下面的.ssh文件夹中找到生成密钥，id_rsa.pub就是我们需要的公钥文件，复制里面所有的内容

  3. 打开github页面

  4. 点击自己头像 选择下面的settings

  5. 进入到settings页面之后左边栏中会有一个 ssh and gpg keys ，点击

  6. 进入到该页面之后，页面的右上角会有一个 New SSH Key 的绿色按钮， 点击

  7. title文本框中选择性（可写可不写）的输入内容

  8. 在key文本框中，粘贴刚才复制的公钥的全部内容

  9. 点击下面的绿色的 Add SSH Key 按钮

  10. 获取或者push项目的时候，一定要记得选择ssh连接，才可以实现免密登录！
